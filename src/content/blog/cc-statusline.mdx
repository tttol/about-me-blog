# できたもの

こんな感じで、ステータスラインに以下の項目を表示できました。

- cc-statusline: 現在のgitブランチ名
- Context: コンテキストウィンドウ
- 5h: 5時間あたり45プロンプトのrate limit
- 7d: 7日あたり40~80時間のrate limit(Pro, Maxプランによって値は変動します)

# statuslineの仕様
Claude Codeのステータスラインとは、チャットボックスの真下に表示される領域のことを指します。この領域はユーザー側でカスタマイズすることが可能で、カスタマイズの詳細内容はシェルスクリプトで定義可能です。


やり方は.claude/settings.jsonに以下を記載して、

```json
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh",
  }
}
```

~/.claude/statusline.shを以下のフォーマットで作成します。

```sh
#!/bin/bash
# Read JSON data that Claude Code sends to stdin
input=$(cat)

# Extract fields using jq
MODEL=$(echo "$input" | jq -r '.model.display_name')
DIR=$(echo "$input" | jq -r '.workspace.current_dir')
# The "// 0" provides a fallback if the field is null
PCT=$(echo "$input" | jq -r '.context_window.used_percentage // 0' | cut -d. -f1)

# Output the status line - ${DIR##*/} extracts just the folder name
echo "[$MODEL] 📁 ${DIR##*/} | ${PCT}% context"
```
